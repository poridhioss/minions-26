Switch Hugging Face Spaces deploy from Docker SDK to free Gradio SDK

Docker SDK on HF Spaces requires a paid hardware tier to keep the
container running 24/7; the free CPU tier forces sleep after 48h.
Gradio SDK stays free forever on the CPU tier, with no Dockerfile,
no port config, and no compose stack.

- app.py: 60-line Gradio front-end (Blocks + Label) that reuses
  app/model_loader.py. Distributes remaining probability mass
  evenly across the other three labels so the UI shows a full
  bar chart.
- README_HF.md: YAML frontmatter switched from
  sdk: docker / app_port: 7860 to sdk: gradio / app_file: app.py
  / sdk_version: 4.44.0. Documented HF_MODEL_ID/HF_TOKEN secrets.
- requirements.txt: pinned gradio==4.44.0 and added
  huggingface_hub>=0.20 (needed by model_loader's HF Hub fallback).
- .dockerignore: now also excludes Dockerfile, docker-compose.yml,
  prometheus.yml, grafana/, scripts/*.ps1, index.html and
  app/main.py since the Gradio Space doesn't need them.

Local docker compose workflow is unchanged - main.py + index.html
still work via docker compose up --build.
